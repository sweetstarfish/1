# 静态资源目录说明

## 目录结构

```
static/
├── css/                    # CSS样式文件
│   ├── common.css         # 通用样式
│   ├── dashboard.css      # 会员空间页面样式
│   └── admin.css          # 管理员页面样式
├── js/                     # JavaScript文件
│   └── dashboard.js       # 会员空间页面脚本
└── uploads/               # 文件上传目录
    └── (用户上传的文件)
```

## 文件说明

### CSS文件

#### common.css
- **用途**: 所有页面共用的基础样式
- **包含**: 基础重置、通用容器、按钮、表单、表格等样式
- **特点**: 响应式设计，适配移动端

#### dashboard.css
- **用途**: 会员空间页面专用样式
- **包含**: 导航栏、好友管理、用户搜索等样式
- **特点**: 现代化的UI设计，交互动画

#### admin.css
- **用途**: 管理员页面专用样式
- **包含**: 仪表板、统计卡片、管理菜单等样式
- **特点**: 专业的管理界面设计

### JavaScript文件

#### dashboard.js
- **用途**: 会员空间页面的交互功能
- **功能**: 用户搜索、好友添加、动态内容更新
- **特点**: 异步加载，用户体验友好

## 使用方式

### 在HTML中引用CSS
```html
<link rel="stylesheet" href="{{ url_for('static', filename='css/common.css') }}">
<link rel="stylesheet" href="{{ url_for('static', filename='css/dashboard.css') }}">
```

### 在HTML中引用JavaScript
```html
<script src="{{ url_for('static', filename='js/dashboard.js') }}"></script>
```

## 设计原则

1. **分离关注点**: HTML负责结构，CSS负责样式，JS负责交互
2. **模块化**: 不同页面使用不同的CSS文件
3. **可维护性**: 样式集中管理，便于修改和维护
4. **响应式**: 适配不同屏幕尺寸
5. **性能优化**: 外部文件可以被浏览器缓存

## 技术特点

- **CSS3**: 使用现代CSS特性（Flexbox、Grid、动画等）
- **ES6+**: 使用现代JavaScript语法
- **响应式设计**: 移动端友好的布局
- **渐进增强**: 基础功能可用，高级功能增强体验
